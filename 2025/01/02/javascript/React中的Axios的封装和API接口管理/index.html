React中的Axios的封装和API接口管理
Javascript hexo
Javascript
Publish Date:   2025-01-02
Read Count:  

当我们面对繁杂的接口的时候，会很烦躁，一个一个的使用Axios去请求，就很感觉很复杂，而使用统一的封装和管理Axios API，有助于提高代码的阅读行，也可以让接口请求变的更加关注业务，而无需每次都考虑接口需要使用什么请求方式。

一、axios封装

在React项目中，涉及到前后端交互的时候，我们通常会使用axios库，它是基于promise的http请求库，可以运行在浏览器和node.js端。它有很多优秀的特性，例如拦截请求和响应、取消请求、转换json、客户端防御XSRF等。如果对axios还不够了解，可以移步axios文档

axios安装

npm install axios

引入封装目录

一般我会在src目录下创建一个专门的service目录，然后在里面创建http.js文件，在这个文件里面引入axiso，并对其进行封装，在创建一个api.js，将各个模块的接口组织到一起，不同的模块可以使用同名的[name].js文件作为各个模块的接口入口

// 引入axios库
import axios from 'axios';
// 引入QS库来处理参数
import QS from 'qs';
// 引入message模块，作为提示弹窗使用
import { message } from 'antd'

环境变量配置

我们的项目可能有不同的环境，包含开发环境、测试环境以及部署环境等。我们通过node的环境变量来匹配相应的url location前缀，axiso.defaults.baseURL可以设置axiso的默认请求地址。

if(import.meta.env.MODE === 'development'){
  axios.defaults.baseURL = 'https://baidu.com';
}else if(import.meta.env.MODE === 'test'){
  axiso.defaults.baseURL = 'https://ceshi.com';
}else if(import.meta.env.MODE === 'production'){
  axiso.defaults.baseURL = 'https://production.com';
}

设置请求超时

可以通过设置axios.defaults.timeout = 1000*10设置默认的请求超时时间。例如，超过了10秒，就会告知用户当前请求超时，请刷新等。

axios.defaults.timeout = 1000*10;

设置请求头

有的时候根据需求，需要为请求添加headers请求头，其格式如下

axios.defaults.headers.post['Content-Type'] = 'application/x-www-form-urlencoded;charset=UTF-8';

请求拦截

对请求进行封装，为什么要对请求拦截呢？例如当一个系统需要登陆之后才能操作的时候，就需要一个token来控制登陆的状态了。

<!-- 请求拦截器 -->
axios.interceptors.request.use(    
    config => {        
        // 每次发送请求之前判断vuex中是否存在token        
        // 如果存在，则统一在http请求的header都加上token，这样后台根据token判断你的登录情况
        // 即使本地存在token，也有可能token是过期的，所以在响应拦截器中要对返回状态进行判断 
        const token = localStorage.getItem('token');        
        token && (config.headers.Authorization = token);        
        return config;    
    },    
    error => {        
        return Promise.error(error);    
})

响应拦截

// 响应拦截器
axios.interceptors.response.use(    
    response => {   
        // 如果返回的状态码为200，说明接口请求成功，可以正常拿到数据     
        // 否则的话抛出错误
        if (response.status === 200) {            
            return Promise.resolve(response);        
        } else {            
            return Promise.reject(response);        
        }    
    },    
    // 服务器状态码不是2开头的的情况
    // 这里可以跟你们的后台开发人员协商好统一的错误状态码    
    // 然后根据返回的状态码进行一些操作，例如登录过期提示，错误提示等等
    // 下面列举几个常见的操作，其他需求可自行扩展
    error => {            
        if (error.response.status) {            
            switch (error.response.status) {                
                // 401: 未登录
                // 未登录则跳转登录页面，并携带当前页面的路径
                // 在登录成功后返回当前页面，这一步需要在登录页操作。                
                case 401:                    
                    router.replace({                        
                        path: '/login',                        
                        query: { 
                            redirect: router.currentRoute.fullPath 
                        }
                    });
                    break;

                // 403 token过期
                // 登录过期对用户进行提示
                // 清除本地token和清空vuex中token对象
                // 跳转登录页面                
                case 403:
                    message({
                        message: '登录过期，请重新登录',
                        duration: 1000,
                        forbidClick: true
                    });
                    // 清除token
                    localStorage.removeItem('token');
                    store.commit('loginSuccess', null);
                    // 跳转登录页面，并将要浏览的页面fullPath传过去，登录成功后跳转需要访问的页面 
                    setTimeout(() => {                        
                        router.replace({                            
                            path: '/login',                            
                            query: { 
                                redirect: router.currentRoute.fullPath 
                            }                        
                        });                    
                    }, 1000);                    
                    break; 

                // 404请求不存在
                case 404:
                    message({
                        message: '网络请求不存在',
                        duration: 1500,
                        forbidClick: true
                    });
                    break;
                // 其他错误，直接抛出错误提示
                default:
                    message({
                        message: error.response.data.message,
                        duration: 1500,
                        forbidClick: true
                    });
            }
            return Promise.reject(error.response);
        }
    }    
});

响应拦截的好处就是服务器返回给我们的数据，我们在拿到之前可以对它进行一些处理，如果返回的状态码为200，则是正常的数据，还可以根据不同的错误状态，进行不同的数据返回和操作。

封装一下get和post方法

/**
 * get方法，对应get请求
 * @param {String} url [请求的url地址]
 * @param {Object} params [请求时携带的参数]
 */
export function get(url, params){    
    return new Promise((resolve, reject) =>{        
        axios.get(url, {            
            params: params        
        }).then(res => {
            resolve(res.data);
        }).catch(err =>{
            reject(err.data)        
    })    
});
}

/** 
 * post方法，对应post请求 
 * @param {String} url [请求的url地址] 
 * @param {Object} params [请求时携带的参数] 
 */
export function post(url, params) {
    return new Promise((resolve, reject) => {
         axios.post(url, QS.stringify(params))
        .then(res => {
            resolve(res.data);
        })
        .catch(err =>{
            reject(err.data)
        })
    });
}

至此，对于axios的封装基本上结束了，使用的时候只需要按照普通模块去实现就好了。
